Why a self-improving Agent为什么需要自迭代 Agent

Traditional iteration depended on multiple teams manually joining dashboards, traces, edited content, bad cases, and experiment history. Diagnosis and experiment design were slow, inconsistent, and difficult to audit.

传统迭代依赖多个团队人工拼接看板、调用链、用户修改、Bad Case 与历史实验，导致问题发现、根因定位和方案设计周期长，且难以审计与复现。

Fragmented evidence证据分散

Funnel metrics, generated output, edits, error codes, and revenue data lived in different systems.

发布漏斗、生成结果、用户编辑、错误码和营收数据分布在不同系统。

Unsafe autonomy自主修改风险

An autonomous Agent cannot be allowed to modify production behavior without explicit scope, versioning, guardrails, and rollback.

自主 Agent 不能在缺乏范围约束、版本管理、实验护栏和回滚机制时直接修改生产系统。

Multi-Agent architecture多 Agent 协作架构

01

Metric & Trace

Unifies Entry UV, Post per UV, listing success, category-level quality, errors, edits, and revenue. Reconstructs requests through ad_id, request_id, and user_id.

统一 Entry UV、Post per UV、发布成功率、分类目质量、错误、编辑和营收口径，并通过 ad_id/request_id/user_id 重建完整链路。

02

Insight & Hypothesis

Turns anomalies and bad cases into structured hypotheses with target cohort, evidence, change scope, expected value, MDE, and rollback conditions.

将异常与 Bad Case 转化为结构化假设，明确目标人群、证据、修改范围、预期收益、MDE 和回滚条件。

03

Development & Guardrail

Limits changes to versioned prompts, category candidates, attribute schemas, model routing, post-processing, and fallbacks, then runs replay, schema, safety, and hallucination checks.

将修改限制在版本化 Prompt、类目候选集、属性 Schema、模型路由、后处理与降级策略，并执行 Replay、Schema、安全及幻觉检查。

04

Experiment & Learning

Designs staged experiments and evaluates conversion, content quality, violation rate, P95 latency, and token cost before storing the outcome as reusable memory.

设计分阶段实验，联合评估转化、内容质量、违规率、P95 延迟和 Token 成本，并将结果沉淀为可复用记忆。

Safety and reproducibility安全性与可复现性

  • Every change binds model and prompt versions, category-tree snapshot, offline replay set, evaluation report, and rollback plan.
  • Release path follows offline validation → shadow → low-traffic canary → staged rollout.
  • Priority combines traffic scale, upside, implementation risk, and similarity to historical experiments.
  • Negative experiments and human review conclusions remain in memory so failed ideas still improve future decisions.
  • 每次变更绑定模型/Prompt 版本、类目树快照、离线回放集、评估报告和回滚方案。
  • 发布路径遵循“离线验证 → Shadow → 小流量 Canary → 分阶段放量”。
  • 综合流量规模、潜在收益、实现风险和历史实验相似度进行优先级排序。
  • 失败实验与人工审核结论同样进入知识记忆，使负向结果参与后续假设生成。

Project value项目价值

Upgraded the AI listing system from manually driven point optimization to a data- and experiment-driven semi-automated evolution loop. The design shortens discovery, diagnosis, and experiment planning while preserving production stability through versioning, guardrails, and rollback.

将 AI 发布从“人工驱动的单点优化”升级为“数据与实验驱动的半自动持续进化”，缩短问题发现、根因定位和实验方案设计链路，并通过版本化、护栏和自动回滚保障线上稳定性。
